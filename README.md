# Adaptive IAM based on Deep Reinforcement Learning

Воспроизводимый программный артефакт магистерской диссертации
**«Адаптивное управление идентификацией на основе искусственного интеллекта: фреймворк для контекстно-ориентированных систем управления доступом»**.

---

## О проекте

Программный артефакт реализует адаптивную политику контроля доступа в форме
марковского процесса принятия решений (MDP) с дискретным пространством действий
{grant, deny, challenge, step_up} и многокритериальной функцией вознаграждения
R = α·R_security + β·R_friction.

Артефакт включает:

- **IAMEnv** – Gymnasium-совместимая среда с генератором пяти сценариев
  (legitimate, behavioral_drift, credential_theft, false_positive,
  novel_attack);
- **Double-DQN-агент** – обучаемая политика на основе Stable-Baselines3;
- **Rule-based baseline и Pure RBAC** – эталонные политики для сравнения;
- **Метрики FAR / TDR / FRR / AFS** – единая функция оценки;
- **Hold-out оценка** на сценарии `novel_attack`, не виденном в обучении;
- **Адверсиальная sanity-проверка** через равномерное аддитивное возмущение
  биометрических признаков;
- **Парето-анализ чувствительности** по коэффициенту α функции вознаграждения.

## Воспроизведение

```bash
# 1. Клонировать репозиторий на тег версии ВКР
git clone --branch v1.0-vkr https://github.com/guseyn1997/adaptive-iam-rl.git
cd adaptive-iam-rl

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate           # Linux / macOS
# .\venv\Scripts\activate          # Windows PowerShell

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Запустить ноутбук
jupyter notebook Obuchenie_agenta_IAMEnv.ipynb
```

Ноутбук исполняется сверху вниз без ручного вмешательства. Полный прогон
(обучение основного агента на 1 000 000 шагов + парето-сканирование по
четырём значениям α) занимает приблизительно 4–6 часов на стандартной
CPU-конфигурации. Результаты детерминированы при фиксированном `seed`.

## Ключевые результаты

| Сценарий | Метрика | DQN | Rule-Based | Pure RBAC |
|---|---|---|---|---|
| credential_theft | AFS | 0.970 | 0.642 | 0.154 |
| credential_theft | TDR | 0.977 | – | – |
| credential_theft | FRR | 0.010 | – | – |
| novel_attack (hold-out) | AFS | 0.945 | – | – |

При парето-сканировании по α зафиксирован резкий фазовый переход FAR между
α = 0.5 (FAR = 0.873) и α = 0.7 (FAR = 0.390) с последующим эффектом
насыщения. FRR остаётся в диапазоне 0.010–0.013 для всех значений α,
что подтверждает устойчивость политики к чрезмерным отказам легитимным
пользователям.

Подробные результаты по всем сценариям приведены в главе 3 ВКР и в каталоге
`results/`.
